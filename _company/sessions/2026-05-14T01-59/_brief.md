# 📋 작업 브리프

**원 명령:** 최신 .net으로 평가시스템 구축해줘

## 요약
평가 시스템 구축은 범위와 기술 스택이 광범위하여 단계적 접근이 필요합니다. 따라서 먼저 비즈니스 목표를 설정하고, 최신 .NET 아키텍처 트렌드를 분석한 후, 구체적인 구조 설계를 진행해야 합니다.

## 분배
- **🔍 Researcher**: .NET 9 또는 그 이상의 버전에서 가장 모범적(Best Practice)인 평가 시스템의 아키텍처 패턴(예: CQRS, Clean Architecture 등)을 조사하고, 이를 적용할 때 고려해야 할 최신 보안 및 확장성 요소를 정리해줘.
- **💼 현빈**: 이 '평가시스템'이 어떤 목적으로 사용될지 가정하여 (예: 성능 평가, 사용자 경험(UX) 평가, 코드 품질 평가 등), 반드시 포함되어야 할 핵심 기능 목록과 이 시스템을 통해 측정하고 수익화할 수 있는 3가지 주요 KPI를 정의해줘.
- **💻 코다리**: researcher가 제시한 최신 아키텍처 패턴과 business가 정의한 KPI 및 기능을 바탕으로, .NET Core 기반의 모듈식 평가시스템 초기 기술 스택(Technology Stack)과 핵심 데이터베이스 구조(Schema Draft)를 설계하고, 가장 먼저 구현해야 할 API 엔드포인트 3개를 정의해줘.
